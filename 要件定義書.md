# 直樹バイク 公式ウェブサイト 要件定義書（2ページ構成版）

**作成日**: 2026-04-14
**バージョン**: 1.0
**対象プロジェクト**: 直樹バイク 公式ウェブサイト（新規）

---

## 1. プロジェクト概要

### 1.1 目的
バイク修理・中古車販売を行う「直樹バイク」の公式ウェブサイトを新規構築する。
店舗情報・サービス内容の発信と、問い合わせ受付を主目的とする。

### 1.2 ページ構成
| ページ | URL | 役割 |
|---|---|---|
| ホーム | `/` | 店舗の全情報を1ページに集約 |
| お問合せ | `/contact` | 問い合わせフォーム + 送信完了 |

### 1.3 対象ユーザー
- 中古バイクの購入を検討している人
- バイクの修理・メンテナンスを依頼したい人
- カスタム・チューニングに興味のあるバイク愛好家

## 3. ホームページ（/）

### 3.1 セクション構成

#### ① ヒーローセクション
- フルスクリーン背景画像（バイク写真）
- キャッチコピー：「確かな技術で最高のバイクライフを」
- サブコピー：店舗の一言紹介
- CTAボタン：「お問合せはこちら」→ `/contact` へ遷移

#### ② 店舗紹介セクション
- 創業からの実績（年数）
- 3つの強み（カード形式）
  - 認定メカニックによる確かな技術
  - 明朗会計・事前見積もり
  - スピーディーな緊急対応
- スタッフ数・メカニック数の表示

#### ③ サービスセクション
- 3つのサービスをカード形式で紹介
  - 修理・メンテナンス（料金目安付き）
  - カスタム・チューニング（料金目安付き）
  - 中古車販売・買取（保証・無料点検付き）
- 各カードに「詳しく問い合わせる」リンク → `/contact`

#### ④ ギャラリーセクション
- microCMSから取得した施工事例写真を表示
- グリッドレイアウト（PC: 3列、タブレット: 2列、スマホ: 1列）
- 写真クリックでライトボックス表示
- カテゴリ：修理・カスタム・中古車販売

#### ⑤ アクセス・店舗情報セクション
- 店舗名・住所・電話番号
- 営業時間（9:00〜18:00）
- 定休日（毎週水曜・第2第4日曜）
- Googleマップ埋め込み
- SNSリンク（Instagram / X / Facebook）

#### ⑥ お問合せCTAバナー
- 「お気軽にご相談ください」メッセージ
- 「お問合せフォームへ」ボタン → `/contact`

## 4. お問合せページ（/contact）

### 4.1 フォーム項目

| 項目 | 種別 | 必須 | バリデーション |
|---|---|---|---|
| お名前 | テキスト | 必須 | 最大50文字 |
| フリガナ | テキスト | 必須 | カタカナのみ |
| メールアドレス | メール | 必須 | メール形式チェック |
| 電話番号 | テキスト | 任意 | 数字・ハイフンのみ |
| 問合せ種別 | セレクト | 必須 | 修理・メンテナンス / カスタム・チューニング / 中古車・買取 |
| お問合せ内容 | テキストエリア | 必須 | 最大1000文字 |
| プライバシーポリシー同意 | チェックボックス | 必須 | チェック必須 |

### 4.2 フォームの流れ
```
入力画面 → バリデーション → 確認画面 → 送信 → 完了画面
```

### 4.3 メール送信（Resend使用）

#### 管理者通知メール
- 宛先：店舗のGmailアドレス
- 件名：【直樹バイク】お問合せが届きました
- 内容：フォーム入力内容すべて

#### 自動返信メール（お客様宛て）
- 宛先：入力されたメールアドレス
- 件名：【直樹バイク】お問合せありがとうございます
- 内容：受付完了のお知らせ + 3営業日以内に返信する旨

### 4.4 完了画面
- 送信完了メッセージ
- ホームへ戻るボタン

## 5. microCMS連携（ギャラリー）

### 5.1 コンテンツモデル：ギャラリー

| フィールド名 | 型 | 説明 |
|---|---|---|
| title | テキスト | 施工タイトル（例：「CB400 エンジンオーバーホール」） |
| description | テキストエリア | 施工内容の説明 |
| image | 画像 | メイン画像 |
| category | セレクト | 修理 / カスタム / 中古車販売 |
| publishedAt | 日時 | 公開日 |

### 5.2 取得方法
- Next.js の `fetch` でビルド時取得（SSG）
- `revalidate` による ISR（更新時に自動再ビルド）

## 7. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | ページ読み込み3秒以内、画像WebP最適化 |
| SEO | メタタグ・OGP・sitemap.xml・robots.txt |
| アクセシビリティ | セマンティックHTML・alt属性・キーボード操作 |
| セキュリティ | HTTPS・CSRF対策・XSS対策 |
| ブラウザ対応 | Chrome / Safari / Firefox / Edge（最新版） |

## 8. 含まない機能（スコープ外）

- ブログ・ニュース機能
- 中古車在庫一覧ページ
- オンライン予約
- 会員登録・ログイン
- オンライン決済
- チャット機能
- SNS投稿の自動埋め込み
