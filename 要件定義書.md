# バイクWebサイト 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名

**直樹バイク 公式Webサイト**

### 1.2 目的

バイク修理・中古車販売業務の情報発信および顧客とのコミュニケーション促進を目的としたWebサイトの構築

### 1.3 対象ユーザー

- 中古バイクの購入を検討している方
- バイクの修理・整備を依頼したい方
- バイクに関する情報を探している方

---

## 2. 機能要件

### 2.1 必須機能

#### 2.1.1 トップページ

- **目的**: サイトの顔となり、事業内容を一目で理解できるページ
- **主要コンテンツ**:
  - メインビジュアル（スポーティなデザイン）
  - 事業紹介（修理・整備、中古車販売）
  - 最新のお知らせ（3-5件表示）
  - 実績ギャラリープレビュー
  - SNSリンク
  - 問い合わせへの導線

#### 2.1.2 会社案内ページ

- **目的**: 会社・店舗の信頼性を伝える
- **主要コンテンツ**:
  - 会社概要（店舗名、代表者名、事業内容）
  - 店舗所在地・アクセス情報
  - 営業時間・定休日
  - 沿革（あれば）
  - スタッフ紹介（あれば）

#### 2.1.3 サービス紹介ページ

- **目的**: 提供サービスの詳細を伝える
- **主要コンテンツ**:
  - バイク修理・整備サービス
    - 一般整備
    - カスタム・チューニング
    - 車検対応
    - 緊急修理
  - 中古バイク販売
    - 在庫車両一覧（写真、価格、スペック）
    - 買取サービス
  - 料金の目安（具体的な金額または価格帯）

#### 2.1.4 実績ギャラリーページ

- **目的**: 施工実績を視覚的に訴求
- **主要コンテンツ**:
  - 修理・カスタム事例の写真
  - 販売車両の写真
  - Before/After形式の表示
  - 各実績の簡単な説明文
- **機能**:
  - カテゴリーフィルター（修理/カスタム/販売車両）
  - 画像拡大表示（ライトボックス）
  - ページネーションまたは無限スクロール

#### 2.1.5 ブログ・お知らせページ

- **目的**: 最新情報の発信と更新頻度の維持
- **主要コンテンツ**:
  - 新着情報
  - イベント告知
  - メンテナンス情報
  - 入荷車両情報
  - バイク関連コラム
- **機能**:
  - 記事一覧表示（サムネイル、タイトル、日付、抜粋）
  - 記事詳細表示
  - カテゴリー分類
  - 日付での絞り込み
  - 関連記事表示

#### 2.1.6 お問い合わせフォーム

- **目的**: 顧客からの問い合わせを受け付ける
- **入力項目**:
  - お名前（必須）
  - フリガナ（任意）
  - メールアドレス（必須）
  - 電話番号（任意）
  - お問い合わせ種別（選択式: 修理依頼 / 中古車購入 / その他）
  - お問い合わせ内容（必須、テキストエリア）
  - プライバシーポリシー同意チェックボックス（必須）
- **機能**:
  - バリデーション（入力チェック）
  - 送信確認画面
  - 送信完了画面
  - メール送信（店舗側と顧客側の両方）

### 2.2 付加機能

#### 2.2.1 SNS連携

- **実装方法**: リンクボタンのみ
- **対象SNS**: Instagram、Twitter（X）、Facebook など
- **配置**: ヘッダー・フッターに固定表示

#### 2.2.2 レスポンシブデザイン

- **対応デバイス**: PC、タブレット、スマートフォン
- **ブレークポイント**:
  - モバイル: 〜768px
  - タブレット: 769px〜1024px
  - デスクトップ: 1025px〜

## 3. 非機能要件

### 3.1 パフォーマンス

- ページ読み込み時間: 3秒以内（目標値）
- 画像の最適化（WebP形式の採用、遅延読み込み）
- Core Web Vitals の基準を満たす

### 3.2 SEO対策

- 適切なメタタグの設定
- 構造化データの実装
- sitemap.xml の生成
- robots.txt の設定
- OGP（Open Graph Protocol）対応

### 3.3 アクセシビリティ

- セマンティックHTML の使用
- 適切な見出し構造（h1〜h6）
- 画像の alt 属性設定
- キーボード操作対応

### 3.4 セキュリティ

- HTTPS通信
- フォームのCSRF対策
- XSS対策
- 個人情報の適切な取り扱い

### 3.5 ブラウザ対応

- Chrome（最新版）
- Safari（最新版）
- Firefox（最新版）
- Edge（最新版）

## 4. 技術仕様

### 4.1 技術スタック

#### フロントエンド

- **フレームワーク**: Next.js 14+ (App Router)
- **ライブラリ**: React 18+
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS または CSS Modules
- **アニメーション**: Framer Motion（必要に応じて）

#### CMS（コンテンツ管理システム）

- **サービス**: microCMS
- **用途**: ブログ・お知らせ、実績ギャラリーの管理
- **管理者**: 非技術者でも更新可能

#### お問い合わせフォーム

- **実装方法**: Next.js API Routes
- **メール送信**:
  - Resend（推奨）
  - SendGrid（代替案）
  - Nodemailer + SMTPサーバー（代替案）

#### ホスティング・デプロイ

- **プラットフォーム**: Vercel
- **特徴**:
  - 自動デプロイ（Git連携）
  - プレビュー環境の自動生成
  - カスタムドメイン対応
  - 無料SSL証明書

### 4.3 外部サービス連携

#### microCMS 設定

- **API エンドポイント**:
  - `/api/v1/blog` - ブログ記事
  - `/api/v1/news` - お知らせ
  - `/api/v1/gallery` - 実績ギャラリー
- **コンテンツモデル**:

**ブログ記事（blog）**:

```
- id: string
- title: string (タイトル)
- content: richEditor (本文)
- thumbnail: image (サムネイル画像)
- category: select (カテゴリー: お知らせ/イベント/メンテナンス情報/コラム)
- publishedAt: dateTime (公開日時)
- createdAt: dateTime
- updatedAt: dateTime
```

**実績ギャラリー（gallery）**:

```
- id: string
- title: string (タイトル)
- description: textArea (説明)
- images: multipleImage (画像、複数枚)
- category: select (カテゴリー: 修理/カスタム/販売車両)
- beforeImage: image (Before画像、任意)
- afterImage: image (After画像、任意)
- publishedAt: dateTime
- createdAt: dateTime
- updatedAt: dateTime
```

## 5. デザイン要件

### 5.1 デザインコンセプト

**スポーティで躍動感のあるデザイン**

### 5.2 カラーパレット（案）

- **プライマリカラー**: レッド系（#DC2626 または #EF4444）
- **セカンダリカラー**: ダークグレー・ブラック系（#1F2937）
- **アクセントカラー**: ホワイト、シルバー
- **背景色**: ホワイト（#FFFFFF）、ライトグレー（#F9FAFB）

### 5.3 フォント

- **日本語**: Noto Sans JP / ヒラギノ角ゴシック
- **英数字**: Roboto / Inter（モダンで読みやすいサンセリフ）
- **見出し**: 太字（700〜900）でインパクトを出す

### 5.4 デザイン要素

- **写真**: 高品質なバイク写真を大きく使用
- **余白**: 適度な余白で視認性を確保
- **角丸**: ボタンやカードに軽い角丸を適用
- **シャドウ**: 控えめなドロップシャドウで立体感
- **アニメーション**: スクロールに応じた要素の表示（パララックス、フェードイン）

### 7.1 コンテンツ更新

- **ブログ・お知らせ**: microCMS 管理画面から随時更新
- **実績ギャラリー**: microCMS 管理画面から随時更新

### 7.2 定期メンテナンス

### 7.3 問い合わせ対応

- フォームから送信されたメールへの返信
- 営業時間内での対応を推奨

## 8. 制約事項・前提条件

### 8.1 実装しない機能

- ✗ オンライン予約システム（ユーザー要望により不要）
- ✗ 会員登録・ログイン機能
- ✗ オンライン決済機能
- ✗ チャット機能
- ✗ SNS投稿の自動取得・埋め込み（リンクのみ対応）

### 8.2 前提条件

- microCMS のアカウント作成が必要
- Vercel のアカウント作成が必要
- メール送信サービス（Resend等）のアカウント作成が必要
- ドメインの取得（必要に応じて）
- 画像素材の提供（店舗写真、バイク写真、ロゴ等）
