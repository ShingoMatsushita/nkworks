## 1. カスタムスキル（Skills）の使い方

### スキルとは？

スキルは、特定のタスクを効率的に実行するための**再利用可能な指示セット**です。プロジェクト固有の作業パターンを標準化できます。

### 作成済みのスキル

#### 📦 react-component

**用途**: Next.js App Router用のReactコンポーネントを作成

**使い方**:

```
スキル: react-component
タスク: Buttonコンポーネントを作成してください
```

**自動的に行われること**:

- TypeScriptでの型安全なコンポーネント作成
- Tailwind CSSでのスタイリング
- 適切なディレクトリへの配置（src/components/）
- レスポンシブ対応
- アクセシビリティ対応

#### 🔗 microcms-integration

**用途**: microCMSのAPIクライアントコードとデータ取得関数を生成

**使い方**:

```
スキル: microcms-integration
タスク: ブログ記事取得のAPIクライアントを作成してください
```

**自動的に行われること**:

- microCMSクライアントの設定
- 型定義の作成
- データ取得関数の実装
- エラーハンドリング
- キャッシュ戦略の適用

### スキルの構造

```
.claude/skills/
├── react-component/
│   ├── skill.json        # メタデータ
│   └── prompt.md         # 実行時の指示内容
└── microcms-integration/
    ├── skill.json
    └── prompt.md
```

### 新しいスキルの作成方法

1. **ディレクトリ作成**

```bash
mkdir -p .claude/skills/my-skill
```

1. **skill.json作成**

```json
{
  "name": "my-skill",
  "displayName": "マイスキル",
  "description": "スキルの説明",
  "version": "1.0.0",
  "author": "作成者名"
}
```

1. **prompt.md作成**

```markdown
# マイスキル

このスキルは〇〇を行います。

## 実行ルール
1. ...
2. ...

## 例
...
```

## 2. カスタムコマンド（Slash Commands）の使い方

### コマンドとは？

Slash Commands（`/command`）は、よく使う操作を簡単に実行できる**ショートカット**です。

### 作成済みのコマンド

#### /setup-project

**用途**: Next.jsプロジェクトの初期セットアップ

**使い方**:

```
/setup-project
```

**実行内容**:

- Next.jsプロジェクトの作成
- 必要なパッケージのインストール
- 環境変数ファイルの作成
- ディレクトリ構造の作成

#### /create-page

**用途**: 新しいページを要件定義書に基づいて作成

**使い方**:

```
/create-page
```

その後、作成したいページを選択します。

#### /check-requirements

**用途**: 要件定義書の内容を確認し、実装状況をチェック

**使い方**:

```
/check-requirements
```

**実行内容**:

- 必須機能の実装状況確認
- 技術要件の確認
- 未実装項目のレポート

#### /review-code

**用途**: コードレビューを実行

**使い方**:

```
/review-code
```

その後、レビューしたいファイルやディレクトリを指定します。

### コマンドの構造

```
.claude/commands/
├── setup-project.md
├── create-page.md
├── check-requirements.md
└── review-code.md
```

各ファイルの先頭には、メタデータを含めます：

```markdown
---
description: コマンドの説明
---

# コマンド名

コマンドの実行内容...
```

### 新しいコマンドの作成方法

1. **.claude/commands/ディレクトリにMarkdownファイルを作成**

```bash
touch .claude/commands/my-command.md
```

1. **内容を記述**

```markdown
---
description: コマンドの簡単な説明
---

# コマンド名

## 実行内容
...

## 手順
1. ...
2. ...
```

1. **使用方法**

```
/my-command
```

## 3. エージェント（Agent）の使い方

### エージェントとは？

Agentは、複雑なタスクを**自律的に**実行する特殊なツールです。私が直接作業するのではなく、専門のAgentに仕事を依頼できます。

### 利用可能なAgent

#### 🔍 Explore Agent

**用途**: コードベースの探索や調査

**使用例**:

```
Explore Agentを使って、要件定義書を詳細に分析してください
```

**得意なこと**:

- コードベースの構造理解
- ファイルの検索
- 複数ファイルにまたがる分析
- ドキュメントの調査

**使用タイミング**:

- プロジェクトの全体像を把握したい
- 特定の機能がどこに実装されているか知りたい
- コードの問題点を発見したい

#### 📋 Plan Agent

**用途**: 実装計画の立案

**使用例**:

```
Plan Agentを使って、ブログ機能の実装計画を立ててください
```

**得意なこと**:

- タスクの分解
- 実装順序の提案
- 依存関係の整理

#### 🛠️ General-purpose Agent

**用途**: 複雑な多段階タスク

**使用例**:

```
General-purpose Agentを使って、セキュリティ対策を実装してください
```

**得意なこと**:

- 複数のステップが必要なタスク
- リサーチと実装を組み合わせたタスク

### Agentの実行方法（私が行う操作）

Agentは、ユーザーが直接呼び出すのではなく、私（Claude Code）が必要に応じて起動します。

**ユーザーがすべきこと**:

```
〇〇について詳しく調べてください
```

**私が行うこと**:

```
Task toolを使って適切なAgentを起動
```

### Agentの利点

1. **自律的な作業**: 複数のファイルを自動で探索
2. **詳細な分析**: 時間をかけて徹底的に調査
3. **専門性**: 各Agentが特定の分野に特化

## 4. 実践例

### 例1: 新しいコンポーネントの作成

**ステップ1: スキルを使用**

```
スキル: react-component
タスク: HeroSectionコンポーネントを作成してください
```

**ステップ2: コンポーネントが自動生成される**

- src/components/features/HeroSection.tsx
- TypeScriptでの型定義
- Tailwind CSSでのスタイリング
- レスポンシブ対応

### 例2: プロジェクトのセットアップ

**コマンドを実行**

```
/setup-project
```

**自動的に実行される**:

- Next.jsプロジェクトの初期化
- 必要なパッケージのインストール
- ディレクトリ構造の作成
- 環境変数ファイルのテンプレート作成

### 例3: コードベースの調査

**Agentを使った調査**

```
現在のプロジェクトで、パフォーマンス改善の余地がある箇所を探してください
```

**Explore Agentが実行**:

1. すべてのファイルをスキャン
2. パフォーマンスに影響する箇所を特定
3. 改善提案をレポート

### 例4: 開発ワークフロー

**1日の開発フロー例**:

```
1. /check-requirements
   → 今日やるべきタスクを確認

2. /create-page
   → 新しいページを作成

3. スキル: react-component
   → 必要なコンポーネントを作成

4. 実装作業...

5. /review-code
   → 作成したコードをレビュー

6. Agentに分析依頼
   → "今日作成したコードのセキュリティをチェックしてください"

7. 修正...

8. Gitコミット
```

## 5. ヒントとベストプラクティス

### スキルの使用

✅ **良い例**:

```
スキル: react-component
タスク: ボタンコンポーネントを作成。primary, secondary, outlineの3つのバリエーションが必要です。
```

❌ **悪い例**:

```
ボタン作って
```

→ 詳細が不足しています

### コマンドの使用

✅ **良い例**:

```
/create-page
```

→ コマンドを実行後、対話形式で詳細を決定

❌ **悪い例**:

```
ページを作成して
```

→ コマンドを使わないと、標準化されない

### Agentの使用

✅ **良い例**:

```
Explore Agentを使って、microCMSとの連携部分を調査し、エラーハンドリングが適切か確認してください
```

❌ **悪い例**:

```
エラーを直して
```

→ どこのエラーか不明確

## 6. トラブルシューティング

### Q: スキルが見つからない

**原因**: スキルのディレクトリ構造が正しくない

**解決策**:

```
.claude/skills/[スキル名]/
├── skill.json
└── prompt.md
```

### Q: コマンドが実行されない

**原因**: ファイル名とコマンド名が一致していない

**解決策**:

```bash
# /my-command を実行したい場合
.claude/commands/my-command.md
```

### Q: Agentの結果が期待と違う

**原因**: 指示が曖昧

**解決策**: より具体的な指示を出す

```
❌ "コードをチェックして"
✅ "src/components/ 配下のコンポーネントで、TypeScriptの型定義が不足している箇所を特定してください"
```

## まとめ

Claude Codeの3つの機能を使い分けることで、開発効率が大幅に向上します：

1. **スキル**: 繰り返し作業の標準化
2. **コマンド**: よく使う操作のショートカット
3. **エージェント**: 複雑なタスクの自律的実行

**次のステップ**:

1. `/setup-project` でプロジェクトを初期化
2. スキルを使ってコンポーネントを作成
3. Agentに分析を依頼

開発を楽しんでください！
